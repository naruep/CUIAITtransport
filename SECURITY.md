# Security Policy

## Reporting a vulnerability

Please do not disclose security vulnerabilities in a public issue.

Report the vulnerability privately to the repository owner through GitHub. Include the affected component, reproduction steps, expected impact, and any suggested mitigation. Do not include real student, staff, or institutional data in the report.

The repository owner will acknowledge the report, assess its severity, and coordinate remediation before any public disclosure.

## Supported versions

Only the current default branch is actively maintained unless a release states otherwise.

## Credential handling

API keys, access tokens, passwords, OAuth credentials, deployment secrets, and personal data must never be committed. If a credential is exposed, revoke or rotate it immediately and remove it from both the current code and Git history.
